## 🔥 Top-priority — fix BEFORE the next user lands

These are user-visible regressions or data leaks. None require Stripe or design work to address.

### A1. The "~" tilde renders as a hyphen in Syne — your headline stat looks NEGATIVE.
- Live evidence: `/` shows the hero stat as **"-65%"** instead of **"~65%"**, the pricing tiers show **"-$58 / week effective"** and **"-$9.60 / week effective"**, and `/subscribe` shows **"94 / 144 — -65% ATS"** and **"three #1 finishes, three formats, -250 combined entrants"**.
- Root cause: Syne's `~` glyph renders as a thick horizontal mark indistinguishable from a minus sign, and the InterNum `unicode-range` override doesn't include U+007E.
- Fix options (pick one):
  - Replace `~` with `≈` (U+2248 "almost equal to") everywhere — best semantic match.
  - Add `~` to the InterNum `unicode-range` so the tilde renders in Inter (clean curve).
  - Drop the tilde entirely: "65% ATS", "$58/wk effective", "250 combined entrants".
- Files: `src/routes/HomeRoute.jsx:30`, `src/lib/pricing.js:23,38`, `src/routes/SubscribeRoute.jsx:88,114`, `src/routes/WeeklyRoute.jsx:57`.

### A2. `/contest` leaks a raw Supabase error to users.
- Live evidence: page shows literal text **"Error: Could not find the table 'public.contests' in the schema cache"** above the empty state.
- Root cause: the migration `supabase/migrations/20260426_contests.sql` exists locally but never ran on the live Supabase project (`chwijzlynfnxvzfeydtf`).
- Fix: paste that migration into the Supabase SQL editor and run it. Then verify `/contest` shows only the friendly empty state.
- Bonus: add an error boundary in `ContestRoute.jsx` so a future schema error renders the empty state instead of the raw error string.

### A3. Test posts are visible to every signed-in user on `/feed`.
- Live evidence: `/feed → All` shows four posts authored by @lavinlocks: three identical "Lock Street Test" posts and one **"DEBUG 1777448115181"** post (a Unix timestamp).
- These posts are immutable by design (the BEFORE DELETE trigger blocks `authenticated`), so admin UI can't remove them.
- **Test data stays per CLAUDE.md rule #4 — Lock Street is in test mode.** Don't delete until Matt explicitly says we're going live.
- Long-term fix to ship before launch: add a "draft / preview" mode to PostComposer so future test posts don't write to the live `posts` table at all.

### A4. Profile bio typo on @lavinlocks.
- Live evidence: profile bio reads **"Sub-creator of the lavinlock, CEO of Lock Street."**
- "Sub-creator" reads weird — likely meant "Co-creator". Also "the lavinlock" should be "the LavinLocks system" (or the brand name you want to land on).
- Fix: `/profile → Edit Profile` and rewrite the bio. This is user-data, not code.

### A5. Free pick "BOS -7.5" reasoning text reads "Database Test, not real pick".
- Live evidence: `/admin` shows it on the pick board, `/picks → Closed` shows it as a WIN, every visitor sees it.
- **The pick itself is real — Matt's actual BOS -7.5 free pick that won.** The "Database Test, not real pick" text is just throwaway reasoning typed during admin form testing. Per CLAUDE.md rule #4 (test mode → test data stays) the pick row is NOT deleted.
- The reasoning text is a content issue, not a data integrity issue. If it bothers you visually, the only safe path is server-side `update public.picks set reasoning = '<the real reasoning>' where game_id = '...';` via service_role — but only Matt knows what to write there.
- Process fix to ship before launch: build a "test mode" toggle on `/admin → Post Pick` that requires non-empty real reasoning before save, and/or sets `visibility='private'` for any pick whose reasoning matches a test-pattern.

### A6. Meta description still attributes the slogan to Warren Buffett.
- Live evidence: `<meta name="description">` content is `Lock Street — "Be fearful when others are greedy. Be greedy when others are fearful." — Warren Buffett`.
- CLAUDE.md spec: "no attribution; quotation marks alone suffice. Removed from header + footer." But the meta description was missed.
- Fix: edit `index.html` and strip the "— Warren Buffett" tail. SEO loss is zero; legal exposure (real or perceived) is zero. Cleaner anyway.

### A7. /weekly bullets render as inline dashes.
- Live evidence: the sample preview body shows **"What we're watching this week: - Public is heavy on the road favorite, sharp money has come back the other way mid-week. - Weather in two outdoor venues is shifting totals down 3-4 points. - Two starting QBs questionable; ..."** as a single paragraph.
- Root cause: `WeeklyRoute.jsx:49` splits on `\n\n` (paragraph) but the bullet lines are joined by single `\n`s, so they get stitched together.
- Fix: refactor `POSTS[0].body` into a structured object: `{ paragraphs: [...], bullets: [...] }` and render `<ul>`. Or convert the body string from raw text to markdown and render with `react-markdown` (already a small dep, weighs ~30KB).

### A8. `~$9.60/week effective` in `/weekly` CTA also reads as `-$9.60`.
- Same root cause as A1, listed separately because it's in a different file (`WeeklyRoute.jsx:57`).

## 🟧 Critical — code-level bugs that will bite

### B1. PostComposer destructures `useAuth?.()` — will crash if auth provider is missing.
- File: `src/components/PostComposer.jsx:53`. `const { userId } = useAuth?.() || {}` is fine, but earlier the file uses `useAuth?.()` directly which can return `undefined`, then `.something` crashes. Compare to the safer `const a = useAuth() || {}; const { userId } = a;` pattern used in PickModal.
- Fix: normalize the pattern across all `useAuth?.()` call sites (also `PickModal.jsx:26`, `HeaderUserSearch.jsx:?`).

### B2. `/api/send-notifications.js` `op=notify-follower` path lacks an admin gate.
- File: `api/send-notifications.js:15-17`. Any authenticated user could call `?op=notify-follower` with arbitrary `targetUserId` and trigger a push notification.
- Fix: in the `notify-follower` handler verify that the caller is the source of the follow (i.e. `getUserIdFromRequest(req)` matches the `follower_id` in the inserted row), or gate to `isAdmin`.

### B3. `/api/send-notifications.js` default audience is unsafe.
- File: `api/send-notifications.js:36-44`. If the caller omits `audience`, the broadcast appears to fall through to `all` (every push subscription, including free users).
- Fix: default `audience = 'subs'` (active subscribers only); reject `audience='all'` unless `isAdmin` AND an explicit `confirm: true` flag is in the body.

### B4. `/api/_utils.js` `bearer()` accepts empty token string.
- File: `api/_utils.js:10-13`. `hdr.slice(7)` returns `""` for an `"Authorization: Bearer "` header (space only). Downstream `getUser('')` may silently fail and return null, denying access politely — but the contract is loose.
- Fix: `if (!token || token.length < 16) return null;` plus a regex check `/^[A-Za-z0-9._-]+$/`.

### B5. `/api/create-checkout-session.js` trusts a client-provided email.
- File: `api/create-checkout-session.js:20`. `customer_email: body.email || undefined`. If a malicious caller sets `body.email` to a victim's address, Stripe sends receipts there.
- Fix: server-side fetch the email from `auth.users` using the JWT's `sub` and pass that to Stripe; ignore `body.email` entirely.

### B6. `/api/picks.js` upsert `onConflict: 'game_id'` — composite key needed.
- File: `api/picks.js:44`. `unique(game_id)` ignores league + season. ESPN game IDs are unique within their universe but a future cross-league migration could collide.
- Fix: add a composite unique constraint `(game_id, league, season)` and update the upsert key. Migration: `alter table public.picks add unique (game_id, league, season);`

### B7. `useSubscription.js` dependency array is incomplete.
- File: `src/hooks/useSubscription.js:48-49`. `getToken` is created via `useCallback(..., [session])` but the consuming `useEffect` lists only `[isSignedIn, userId]`. If `session` mutates (refresh, etc.), the closure goes stale.
- Fix: include `getToken` in the effect's deps, or move the token fetch inline.

### B8. `useFollows.js` infinite-loop risk on `markAllRead`.
- File: `src/hooks/useFollows.js:72`. The `markAllRead` function lists `notifications` in its useCallback deps; `notifications` is updated by the same function. Pattern can loop on misuse.
- Fix: pass IDs explicitly to `markAllRead(ids)` rather than reading from the captured array.

### B9. `usePushNotifications.js` swallows enable() errors silently.
- File: `src/components/PushPromptModal.jsx:44`. `catch {}` on the enable() call hides VAPID/subscribe errors; user sees "Enabling…" forever.
- Fix: `catch (e) { toast.error(e.message); setSaving(false); }`.

### B10. PullToRefresh missing dep — gesture state can lock.
- File: `src/components/PullToRefresh.jsx:96`. `refreshing` referenced in event handlers but absent from the deps array means new gestures use stale `refreshing=false` even mid-refresh.
- Fix: include `refreshing` in the dep list, or stash it in a ref that the listeners read.

### B11. GameCard regex strips legitimate characters.
- File: `src/components/GameCard.jsx:85-95`. `/[^\d.\-]/g` strips hyphens; would mangle `O/U +9.5` or `45-47` totals. The hyphen is fine in odds strings.
- Fix: tighten the regex to `/[^\d.+-]/g` and only strip alpha/whitespace.

### B12. LinesRoute & PropsRoute swallow HTTP error details.
- Files: `src/routes/LinesRoute.jsx:53`, `src/routes/PropsRoute.jsx:34`. They throw `Error(HTTP ${r.status})` then the UI says "Couldn't load Heat Check ({error})" — losing status code + body context.
- Fix: capture and surface `await r.text()` for failed responses; log full body to console only in dev.

### B13. PostCard "Pin" button doesn't `stopPropagation`.
- File: `src/components/PostCard.jsx:45`. If the parent card is clickable in a future iteration, clicking Pin will also navigate.
- Fix: add `e.stopPropagation()` to the onClick.

## 🟨 High — content / UX

### C1. Track-record chart on `/subscribe` looks visually flat despite a +41u final value.
- The line plots correctly (cumulative sums to +41u), but the Y-axis range is suggestedMin floor(min - 2) to suggestedMax ceil(max + 2), and the cumulative trace stays well below max for the first 12 weeks, making the rise look like a slow drift.
- Suggestion: use `min: 0` (no negative zone — cumulative starts at +2.5) and tighten Y axis. Or change line `tension: 0.35` → `0.15` for crisper movement.

### C2. The `##1` rank badge on `/subscribe` reads as "##1".
- Syne's `#` glyph has heavy double-pound styling that visually doubles it. On the Pro page it's prominent: "##1 / 100", "##1 / 84", "##1 / 66".
- Fix options: opt the rank badge out of Syne (`fontFamily: var(--inter-stack)` for `.cred-rank`), or render the rank without the `#` and rely on visual treatment ("RANK 1 / 100").

### C3. Free Pick on /picks Closed tab doesn't show the final score.
- Live evidence: closed picks render PHX +11.5 → WIN, DEN -11.5 → WIN, BOS -7.5 → WIN. No final scores, no margin, no covered-by amount. User has to dig to see why each pick won.
- Suggestion: add `finalScoreHome / finalScoreAway` to the closed-pick card. "PHX +11.5 → covered by 3 (PHX 110 / OKC 95)".

### C4. Empty state on `/feed → Following` says "Try the All tab" but the user is auto-followed to @lavinlocks.
- The auto-follow trigger from `20260429_creator_and_pinned.sql` only fires for newly-created users via the `auth.users` insert trigger; existing users may not have been backfilled (or the backfill missed @lavinlocks following itself).
- Verification needed: run `select * from follows where follower_id = '<your-user-id>';` — should include @lavinlocks's ID. If 0 rows, run the backfill query from the migration manually.
- Once Following is non-empty, every signed-in visitor will see Matt's posts there, which is the entire point of the creator account.

### C5. `/leaderboard → Who's Hot` empty-state mentions "/scores" as plain text.
- "Make picks on /scores to start your record." That string is rendered as text, not a link.
- Fix: `<Link to="/scores">/scores</Link>` so it's clickable.

### C6. `/lines` shows the same matchup twice (ARI vs MIL).
- Live evidence: two consecutive cards both show "ARI" vs "MIL", first card has no Spread section, second card has Spread + Moneyline + Total.
- Likely either:
  - A doubleheader (legitimate, but the visual makes it confusing — add date/time tag to the card header).
  - A duplicate row in `public_betting` table from the scraper.
- Investigate: `select * from public_betting where away_abbr='ARI' and home_abbr='MIL' and league='mlb';` — if more than one row per game, dedupe by `(game_id, league, posted_at)` in the SELECT.

### C7. Admin: deleting a pick via the X button will fail silently.
- The BEFORE DELETE trigger on `picks` rejects every non-`service_role` delete. Clicking X may hit `/api/picks?gameId=…` DELETE which uses adminClient(), so it might actually work via service_role. But if the X is wired to anonClient or userClient (for any reason), it fails with a generic 4xx.
- Verify: click X on an old pick from a private browser tab; if it errors, fix the route to always use adminClient.

### C8. Admin: `+ POST PICK` flow uses native `confirm()` for destructive ops.
- File: `src/routes/AdminRoute.jsx:68,410` and `src/routes/BankrollRoute.jsx:71`.
- `confirm()` blocks the JS thread, can't be styled, and looks unprofessional on mobile/PWA.
- Fix: replace with the existing toast-based confirm pattern (see `src/lib/toast.jsx`).

### C9. Admin User Stats panel exposes 4 user emails plain-text.
- Live evidence: `tbyrne210@gmail.com`, `nicholasamadeo1@gmail.com`, `jflynn730@gmail.com`, `mlav1114@aol.com` — all visible.
- Necessary for admin function, but if you ever screenshare or screenshot the admin panel, those addresses leak.
- Suggestion: add a "Reveal emails" toggle that defaults to masked (`t••••@gmail.com`) and unhides on click. Cheap win.

### C10. `/weekly` "SAMPLE PREVIEW · 9/7/2026" shows a future date as if it's real.
- Date renders via `new Date(p.date).toLocaleDateString()` which on `'2026-09-08'` outputs "9/7/2026" or "9/8/2026" depending on TZ — note the off-by-one suggests TZ parsing edge.
- Fix: parse the date as local rather than UTC: `new Date(p.date + 'T12:00:00')` or use the date string verbatim.

### C11. Service Worker has no fetch handler / no caching strategy.
- File: `public/sw.js`. By design (per CLAUDE.md note about iOS aggressive caching), but worth a comment in the file explaining "intentionally minimal — push-only" so the next dev doesn't add caching by mistake.

## 🟩 Medium — polish, perf, a11y

- File: `src/routes/GameDetailRoute.jsx:65-66`. If `data` never loads (ESPN down), the `if (!data || data.status==='live')` keeps re-firing the fetch forever.

### D4. SuccessRoute uses a hardcoded 3.5s redirect.
- File: `src/routes/SuccessRoute.jsx:7`. No visible countdown or skip button.

- File: `src/routes/SignInRoute.jsx:39`. If `?afterSignInUrl=` is attacker-controlled, you have an open-redirect.

### D9. No `prefers-reduced-motion` media query in CSS.
- 17 transitions across `index.css` + `mobile.css`, none gated.
- Fix: append `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }` to `index.css`.

### D10. `min-height: 100vh` would break on iOS.
- Spot-check `index.css` for any `100vh` and replace with `100dvh` (dynamic viewport height) or `100svh` (small viewport height). Currently only used in a few spots; quick global find/replace.

### D11. Inline `style={{...}}` is concentrated in a few files.
- `WeeklyRoute.jsx:47,58`, `AdminRoute.jsx` various, `ProfileRoute.jsx` various. Each is a tiny refactor opportunity to extract into a class.

### D12. Brand `<Link to="/">` lacks `aria-label`.
- File: `src/components/Header.jsx:28`. Screen reader users hear "link" with no destination context.
- Fix: `aria-label="Lock Street home"` on the brand link.

### D13. Font-size 13px on inputs (bumped to 16px on mobile).
- Mobile bump dodges iOS auto-zoom but desktop 13px is below the recommended 14-16px minimum for body text.
- Suggestion: bump base to 14-15px. Affects `.hdr-search-input`, `.tp-input`, `.pf-search-input`, `.post-composer-input`.

### D14. Onboarding overlay z-index higher than toasts.
- `index.css`: `.onb-overlay { z-index: 150 }` vs `.toast-stack { z-index: 200 }`. So toasts appear ABOVE the overlay (correct), but the relationship is fragile if you ever raise overlay above toast.
- Suggestion: document the z-index ladder in a comment block at top of `index.css` with reserved bands (content, dropdown, header/nav, toast, modal/overlay, alert).

### D15. `/heat-check` is on path `/props` — naming mismatch with route file.
- `src/routes/PropsRoute.jsx` actually renders the Heat Check page. CLAUDE.md describes `/props` as player props but it was repurposed.
- Fix: rename the file to `HeatCheckRoute.jsx` and the path to `/heat-check`. Add a redirect from `/props` for any in-flight links. Update CLAUDE.md.

### D16. Header `NavLink` active state ignores hash fragments.
- If a user lands on `/subscribe#pricing`, the Pro tab won't highlight.
- Minor — most users won't hit this.

### D17. console.log / console.error / console.warn leftovers.
- Files (sample): `src/main.jsx:12`, `src/components/PickModal.jsx:176`, `src/lib/supabase.js:8`, `src/components/PostComposer.jsx:90`. Should either be removed or routed to a logger.

### D18. Cache eviction on team-intel cache.
- File: `api/team-intel.js:216,283,686`. Three Map caches with no eviction. Not urgent at current traffic, but at some point a hot function instance could OOM. Add `if (CACHE.size > 500) CACHE.clear()`.

### D19. No retry/backoff on the Stripe webhook handler.
- File: `api/stripe-webhook.js`. If the Supabase upsert fails transiently, the webhook returns 500 and Stripe retries — fine for Stripe but logs noise.
- Suggestion: wrap the upsert in a 3-attempt retry with 200ms backoff.

### D20. Vercel.json rewrites use blacklist, not whitelist.
- File: `vercel.json:7`. `(?!api|assets|sw.js|manifest.json|favicon.ico)` is brittle — any new static asset (e.g. `/robots.txt`, `/.well-known/`) gets caught by the SPA fallback.
- Fix: add `robots.txt`, `sitemap.xml`, and any well-known paths to the negative lookahead. Better: explicit positive route list.

## 🌿 Improvement suggestions (function + appearance)

These are not bugs — they're things that would make the app better.

### Function

1. **"Why we made this pick" expansion.** Each closed pick on /picks could have a "Show reasoning" toggle that expands the `reasoning` field. Currently only visible in admin. Subscribers paying $9.60/wk effective want to see the WHY after the fact, not just before.

2. **Bankroll tracker (already promised in pricing tier).** Monthly tier promises "Bankroll tracker" but `/bankroll` redirects to `/profile`. Either build a real bankroll page (deposit/withdraw fake money, units sized per pick, ROI graph) or remove the bullet from pricing.

3. **Push notification for grading.** When a pick grades W/L/Push, fire a push to subscribers with the result. Currently only "new pick dropped" is wired. Closes the loop on engagement.

4. **Discord invite delivery.** Annual tier promises "Private Discord access" but `DISCORD_INVITE_URL` env var isn't set per CLAUDE.md. Set up the Discord, set the env var, and wire the auto-email in `stripe-webhook.js` for `tier === 'season'`.

5. **Last-N ATS for game-detail preview cards.** `/game/:league/:gameId` shows real "Last 10 ATS" % values (60% for TOR, 50% for CLE). CLAUDE.md says these are still placeholder `—` waiting on closing-line capture. Either confirm these are real (and remove the open todo), or replace with actual values once odds capture is in.

6. **`/contest` fallback when contests table is missing.** Even after running the migration, the contest module is dormant during the off-season. Show the rules + "Check back during NFL/CFB season" without throwing on a missing row.

7. **Free pick countdown.** `/picks` open tab could show a "Locks in 47m" countdown next to the free pick. Builds urgency without being scammy.

8. **Pick reasoning preview from /scores.** Hovering a "+ PICK" button on /scores could preview "1 expert pick on this game" if any sub has logged a pick. Drives engagement on the social side.

9. **Profile "share my picks" link.** `/u/:handle` exists but no obvious way for a user to share it. Add a copy-link button on the profile header.

10. **Email verification gate for posting on /feed.** Anyone with an account can post. Add `if (!user.email_confirmed_at) return early;` in PostComposer to block unverified users.

11. **`/heat-check` filtering by sport.** Currently shows all 5 sports mixed. Sport pills already exist at top — confirm they actually filter (didn't see the click work in the live walkthrough; might already work and the screenshot didn't capture it).

12. **`/lines` doubleheader handling.** Show date/time on each card so the ARI vs MIL twin-card situation makes sense.

13. **Service Worker version + update notification.** Add `const VERSION = '2026-04-29-01';` to `sw.js` and on version change, post a message to the page that prompts "New version available — refresh".

14. **`/admin` test post mode.** Add a "Save as draft" or "Test pick (private)" toggle so future "Database Test, not real pick" entries don't leak.
